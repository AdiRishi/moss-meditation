# Zen Design

> **Status: draft preparation checkpoint.** No final UI boards or brand deck have been generated yet. The written system and screen inventory below are working proposals for review, not approved product decisions.

This directory is the future canonical design handoff for Zen. It translates the product intent in [`VISION.md`](../VISION.md) and [`PRODUCT.md`](../PRODUCT.md) into a visual system and screen set.

## Provenance

| Artifact | Source | Current status |
| --- | --- | --- |
| `VISION.md` and `PRODUCT.md` | The product direction described by the project owner | Drafts awaiting review and commit |
| `BRAND.md` | Prior brand deck, product documents, and editorial synthesis | Draft; includes proposed implementation details |
| `SCREENS.md` | Product documents, prior screen boards, and product-design synthesis | Draft; proposed surface inventory and navigation |
| `tokens.json` | Prior deck values translated into machine-readable tokens | Draft; exact implementation mapping is not approved |
| `assets/zen-enso.png` and `assets/zen-app-icon.png` | Approved production app assets | Existing source assets copied for handoff |
| `screens/` and `brand-deck/` | Intended ImageGen and presentation outputs | Not generated |

## Contents

- [`BRAND.md`](./BRAND.md) — draft identity, colour, typography, layout, motion, sound, voice, and accessibility specifications.
- [`SCREENS.md`](./SCREENS.md) — proposed product-surface inventory and interaction intent.
- `assets/` — production-ready identity and visual assets used by the design system.
- `screens/` — high-fidelity UI boards covering every planned app surface.
- `brand-deck/` — editable presentation, PDF, and rendered slides for the full brand and product design system.

The assets in `previous-design-work/` remain the selected visual reference until this design package is reviewed and explicitly approved for replacement.
